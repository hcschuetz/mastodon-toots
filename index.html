Mastodon

Test: Embedding Mastodon Posts

The column on the left contains recent toots from a mastodon account. This is implemented by a web component using the Mastodon REST API.

The component provides attributes for configuring the toot retrieval:

  • The Mastodon instance and the account id of the originator
    (Using the numeric id instead of the username saves one roundtrip for looking up the id.)
  • Flags to omit replies or boosts
  • An optional limit for the number of retrieved toots, defaulting to the instance's default limit

The output of the component does not go to shadow DOM. So it can be tweaked using CSS styling for the main ("light") DOM tree.

The component implementation can be used out of the box if it fits your needs, but is also easy to adapt/extend.

Features that might be added:

  • Smarter sanitizing of the HTML toot contents
  • Support for attached media beyond images
  • Support for custom emojis (represented as :emoji_name:; API docs here)
  • Preview referenced web pages
  • Regular or user-triggered updates for displaying new posts
  • Internationalization/localization of error messages
  • Support for attribute changes (such as showing a different account)
